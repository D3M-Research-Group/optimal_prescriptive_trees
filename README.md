# README

**Manuscript title**: Learning Optimal Prescriptive Trees from Observational Data

**Authors**: Nathanael Jo, Sina Aghaei, Andrés Goméz, Phebe Vayanos

# 1. Overview
This directory replicates the results from our paper. There are three sub-directories: 1) `data/` contains all the raw and processed data used in the experiments; see Sections 2 and 3 of this README for further details. 2) `results/` stores the results from all methods and experiments conducted. 3) `scripts/` contains all the code relevant to running the methods, process the data, and visualize the results similar to the figures and tables in the paper; see Section 5 of this README for further details.

# 2. Data availability and provenance

This paper relies on data, and all data necessary to reproduce the results of the paper are included. Below is a detailed description of how the data was obtained, allowing to reproduce the dataset. The following describes the structure of our data:

```
data
├── processed
│   ├── synthetic
│   │ 	├── data_{train/test}_{p}_{s}.csv: p is the probability of correct treatment assignment, while s ∈ {1, ..., 5} are seeds controlling different splits
│   │ 	├── data_{train/test}_enc_{p}_{s}.csv: same as above, but covariates are discretized
│   ├── warfarin
│   │ 	├── seed{e}: e ∈ {1, ..., 5} is the seed to generate the treatment randomization
│   │ 	│ 	├── data_{train/test}_{p}_{s}.csv: p is the randomization condition, while s ∈ {1, ..., 5}
│   │ 	│ 	├── data_enc_{train/test}_{p}_{s}.csv: same as above, but covariates are discretized
│   ├── warfarin_intermediate: warfarin dataset prior to splitting
│   │ 	├── seed{e}
│   │ 	│ 	├── warfarin_{p}.csv: p is the randomization condition
│   │ 	│ 	├── warfarin_enc_{p}.csv: p is the randomization condition
├── raw
│   ├── warfarin.csv
```


### a) Synthetic data

Our data generation process is adapted from [Athey and Imbens (2016)](https://www.pnas.org/doi/10.1073/pnas.1510489113), see Equation (10) in the paper. See #5 below for a description of the code to generate this data. When the code is executed, files are created in `data/processed/synthetic/`. As described in the structure above, the data are also split into train/test, as well as by randomization condition (p) and using 5 different splits (s). This data will be the basis for training our method and the various baseline methods in the paper.

### b) Warfarin dosing data
We use a publicly available dataset on warfarin dosing collected by the [International Warfarin Pharmacogenetics Consortium (2009)](https://www.nejm.org/doi/full/10.1056/NEJMoa0809329), where counterfactuals are available. This allows us to simulate settings with different observational experiments: a fully randomized trial (p = 0.33), and a more informed treatment policy that performs better than random (p = r0.06, 0.11). See Section 6.1 for details. 

The raw dataset can be found in `data/raw/warfarin.csv`. See #5 below for a description of the code to process this data. In short, the code first cleans the data and generates the randomization policies as seen in `data/warfarin_intermediate/seed{e}`. Each e governs a different seed to generate the treatment policy. Then, the code splits this data into train/test splits as seen in `data/warfarin/seed{e}`.

Note that for both datasets, we also create a discretized version where there exist continuous covariates. This is denoted by `data_enc_`.

#### Covariates and outcomes

| Variable | Description |
|--------|-------------|
| V1 | Randomly generated standard normal covariate |
| V2 | Randomly generated standard normal covariate |
| t | Treatment assignment (0 = control, 1 = treatment) |

These variables are learned from the training data and passed into the methods used in the paper.

| Variable | Description |
|--------|-------------|
| prob_t_pred_log | Estimated propensity score P(t | X) using logistic regression |
| prob_t_pred_tree | Estimated propensity score using a decision tree |
| lasso0 | Lasso regression predicting counterfactual outcome for treatment t = 0 |
| lasso1 | Lasso regression predicting counterfactual outcome for treatment t = 1 |
| linear0 | Linear regression predicting counterfactual outcome for treatment t = 0 |
| linear1 | Linear regression predicting counterfactual outcome for treatment t = 1 |

- scikit-learn=1.7.2

All approaches used 6 CPUs, each having 4 GB of memory. All of the MIO-based approaches use the Gurobi solver (here, version 10.0.0). Most of the methods run in less than 1 minute, except for K-PT, B-PT and DR (d > 2), which can require up to 4-5 hours to solve to optimality. Specific runtimes can be found in Figures 4 and 6, as well as in Appendix EC.5.


# 5. Programs/Code
The following describes the structure of our scripts:
```
scripts
├── methods
│   ├── CF: causal forest method
│   ├── Kallus_Bertsimas: K-PT and B-PT MIO methods
│   ├── OPT: Our method
│   ├── policytree: policytree method
│   ├── RC: Regress and compare method
├── process_data
│   ├── synthetic: code to generate and process synthetic data
│   ├── warfarin: code to process warfarin dosing dataset
├── viz
│   ├── fig#.py
│   ├── table#.py
```

Code in python (`.py`) can be executed as is given the required packages. Code in R (`.R`) reads in a text file `global_file_path.txt`. Users should fill in the absolute file path to the current directory before executing code in R.

### Methods
##### CF (Causal Forest and Causal Trees)
There are two R scripts, one for running the synthetic experiments and one for warfarin. By default, they run the causal forest method and save results into the `results/` directory. See instructions within the R script to train causal trees instead.

##### Kallus_Bertsimas (B-PT and K-PT methods)
The python files work together to run the B-PT and K-PT methods. `logger.py` is helper code to lof the results. `main.py` is the driver. `Primal.py` defines the variables and constraints that make up the MIO. `Tree.py` defines the tree structure. In order to run the experiments, we provide python scripts in `slurm/bash_generator_{K-PT/B-PT}.py`. These script generate slurm bash files (examples also provided in the `slurm/` directory) that run all our experiments. There is some customization required depending on the modules and python environment used to run Gurobi, see the `TODO` in the python script.


##### OPT (Our proposed method)
The structure is similar to the B-PT and K-PT methods. The only difference is we additionally have `main_agg.py` and `Primal_agg.py`, which implement the more efficient, aggregated version of our method, see first paragraph of page 11. The `slurm/` directory also contains python scripts to run the fairness and budget constraint experiments. Again, pay attention to the `TODO` in the python script, depending on the modules and python environment one uses to run Gurobi.


##### policytree
There are two R scripts, one for running the synthetic experiments and one for warfarin. The scripts will save results into the `results/` directory.

##### RC (Regress and compare)
There are two python scripts, one for running the synthetic experiments and one for warfarin. The scripts will save results into the `results/` directory.


### Process_data
In each sub-directory, scripts are numbered by the order in which they should be executed. For `synthetic/`, the first script generates the synthetic data and creates the relevant train/test splits, while the second scripts trains various models to predict counterfactual outcomes. For `warfarin/`, the first script processes the raw dataset, then the second script splits the data into train/test splits, and the last script predicts counterfactual outcomes.


### Viz
Each `fig#` script and `table#` script generates a figure and table, respectively. For example, `fig4.py` will generate `./figs/4.pdf`, while table scripts prints out LaTeX. All figures are already in the directory corresponding to the numbering in the paper.
